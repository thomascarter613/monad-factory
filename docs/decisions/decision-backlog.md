# Decision Backlog

The following ADRs should be created during the Product Definition and Implementation Planning phase.

## Initial ADR Set

1. Build Monad OS as an SDLC control plane. Accepted in ADR-0001.
2. Use Rust for the CLI core. Accepted in ADR-0002.
3. Use `workspace.toml` as the canonical manifest. Accepted in ADR-0003.
4. Wrap native tools rather than replacing them. Accepted in ADR-0004.
5. Use Fumadocs for documentation. Accepted in ADR-0005.
6. Design for AI, cloud, and database agnosticism. Accepted in ADR-0006.
7. Support Nx under the hood. Accepted in ADR-0007.
8. Support Buck2 and Pants as enterprise build backends. Accepted in ADR-0008.
9. Keep the local core functional without SaaS. Accepted in ADR-0009.
10. Design for future hosted SaaS control plane. Accepted in ADR-0010.
11. Use policy packs for governance extensibility. Accepted in ADR-0011.
12. Use packs/plugins for ecosystem extensibility. Accepted in ADR-0012.
13. Treat evidence as a first-class artifact. Accepted in ADR-0013.
14. Treat lifecycle graph as the core product moat. Accepted in ADR-0014.
15. Require human approval gates for risky AI actions. Accepted in ADR-0015.
